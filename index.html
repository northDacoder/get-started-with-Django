Before we get started

Before installing Django, we want to setup a few things that will save us major headaches down the road. Installing multiple packages directly into your project runs some serious risks of dependency issues as your project evolves.

As you start to build multiple web applications, each relying on different versions of different packages, things can quickly get out of whack. This is especially true for those that are not taking advantage of using virtual environments. This is usually the point in which you google your brain out until StackOverflow recommends to add a virtual environment to your project workflow.

Even though there is value to learning from your own mistakes, this is on that is a better lesson to learn before starting. Utilizing virtualenvwrapper & vitualenv will save you huge headaches down the road.

What do you mean by virtual environment?


A virtual environment is a way to create isolated Python environments. This solves the problem of dependencies, versions, & indirect permissions. Different projects will require different dependencies & it is best to keep them separate so you are only working on one isolated Python environment at a time.


Step 1

To install both virtualenv & virtualenvwrapper run the following command in the shell of your choice

$ pip install virtualenvwrapper

After you've successfully installed virtualenvwrapper & virtualenv add the following lines to the startup file for whatever shell (terminal) you are using


Step 2

export WORKON_HOME=$HOME/.virtualenvs
export PROJECT_HOME=$HOME/directory-you-do-development-in
source /usr/local/bin/virtualenvwrapper.sh

Close out of your shell & re-open. This will load everything you need to get started using virtualenvwrapper


Step 3

Let's create our first virtual environment by running mkvirtualenv along with a name for this virtual environment

$ mkvirtualenv DJANGOVIRTUALENV1

In this example I named my virtual environment "DJANGOVIRTUALENV1", but you can name yours whatever you want.


Step 4

This should prepend the name of your virtual environment in parenthesis at the beginning of your shell

(DJANGOVIRTUALENV1)

To stop your virtual environment, run the command deactivate


Step 5

Now we are going to install a copy of Django that

$ pip install django

This will install the latest version of Django into our virtual environment, isolated from other installations or users on our computer

To double-check that we have successfully installed the latest version of Django, run the command below

$ which django-admin.py

It should return the path to your installation

/usr/local/bin/django-admin.py

Step 6

Now that we are working in our virtual environment we can run the django-admin.py startproject command & give your project a name 

$ django-admin.py startproject django-starter

This is going to create the project structure for a new django project

django_project/
    manage.py
    django_project/
        settings.py
        urls.py
        wsgi.py

To briefly explain what these files are doing:

settings.py is just a python script that will manage all of the settings & configurations for our project

As your project grows in size or at the point of launching it onto a hosted url, you will want to start separating the settings.py file into different files for your different environments

django_project/
    manage.py
    django_project/
        settings/
            production.py
            local_settings.py
            settings.py
            development.py
            team.py

This will allow you to keep different settings when running your project locally on your localhost or local webserver & production settings when you are deploying your application to something like Heroku, AWS, Docker, etc.
